# 印出來顏色不對:先查這裡

> **這一頁是用一整個下午換來的。** 曾經為了「印出來顏色比較淡」,
> 依序試了:換 ICC profile、補 OutputIntent、提升 PDF 版本、換影像編碼、
> 改解析度、甚至直接把參考檔的 CMYK 像素挖出來當底圖。
> **全部沒用**——因為那台印表機的驅動是 PCL XL,**根本沒在走 CMYK 那條路**,
> 所有色彩設定在送到印表機前就被丟掉了。
>
> 真正的解法是把驅動的 PDL 從 PCL XL 改成 KPDL(PostScript),五分鐘。
>
> **所以:使用者說「印出來顏色不對」時,不要先動檔案。**

---

## 第 0 步:30 秒判定「是不是檔案的問題」

**把同一份檔案拿到另一台電腦印。**

| 結果 | 結論 |
|---|---|
| **兩台印出來不一樣** | **檔案沒問題**,是電腦/驅動。不要再碰檔案了,往下走。 |
| 兩台印出來一樣(但都不對) | 才有可能是檔案,或是兩台設定相同。 |

這一步幾乎不花時間,卻能直接砍掉一半的可能性。**先做這個,再做任何事。**

## 第 1 步:問這三個問題

1. **印表機型號?**
2. **從什麼軟體列印?**(Adobe Reader / Chrome / Edge…)
3. **驅動的 PDL 是什麼?**(PCL XL / PCL5 / PostScript)

第 3 題是關鍵,而且使用者通常不知道答案。下面教怎麼看。

## PDL 決定一切

**PDL(頁面描述語言)是 PDF 內容真正被送進印表機的形式。** 不同 PDL 是完全不同的成像路徑:

| PDL | 色彩能力 | 你的 CMYK/ICC/OutputIntent |
|---|---|---|
| **PCL XL (PCL6)** | **RGB only**,無 ICC 色彩管理 | **全部被丟掉**。驅動先把一切轉回 RGB,再由印表機自行轉成碳粉的 CMYK。 |
| PCL5c | 有限 | 大多丟掉 |
| **PostScript / KPDL** | 完整,支援 DeviceCMYK 與 ICC | **會生效** |

**在 PCL XL 上做 CMYK 是負面效果**:多了一次「sRGB → CMYK → 轉回 RGB」的來回,飽和度被壓縮兩次,顏色反而更沉。**那台機器上該送 RGB。**

### 怎麼看目前是哪個 PDL(兩個方法)

**方法 A — Adobe 的進階列印設定(最快,不用翻設定)**

列印對話框 → **「進階(D)」**按鈕(在「內容(P)」旁邊):

- 最上面**有「PostScript 選項」**區塊(語言等級 2/3、字型和資源策略…)→ **PostScript**
- **沒有**那一區 → **不是 PostScript**(PCL)

**方法 B — 驅動視窗的角落**

Kyocera KX 驅動的預覽窗格右下角會印目前的 PDL 標記(例:`PCL XL`)。

### 怎麼改(Kyocera KX 為例)

```
設定 → 藍牙與裝置 → 印表機與掃描器
→ 點該印表機
→ 「印表機內容」        ← 注意:不是「列印喜好設定」!
→ 「裝置設定」頁籤
→ PDL → 改成 KPDL → 確定
```

**「列印喜好設定」和「印表機內容」是兩個不同的視窗**,PDL 只在後者。使用者找不到 PDL 時,九成是開錯視窗。

## 其他每台電腦會不同的東西

同一份檔案在兩台電腦印出來不同時,依可能性排序:

1. **PDL**(PCL XL vs PostScript)← 影響最大
2. **驅動版本**(頁籤數量不同就是版本不同的線索)
3. **Adobe 列印對話框 → 進階(D) → 「以影像方式列印」** — 打勾會把整頁光柵化,顏色變淡且出現顆粒
4. **Adobe 進階(D) → 色彩管理** — 「讓印表機決定色彩」「保留 CMYK 原色」等
5. **Windows 替印表機關聯的 ICC 描述檔**
   (查:`HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ICM\ProfileAssociations`,
    或 印表機內容 → 色彩管理)
6. 驅動設定:EcoPrint、列印品質、顏色再製、節省碳粉

### 最快的搬移法

Kyocera KX 驅動有**設定檔**功能:在「正常」那台按 **「儲存為(A)…」**匯出,
在「不正常」那台按 **「設定檔(L)…」**載入。連你沒注意到的設定都會一起搬。

## 縮放也會影響

Adobe 列印對話框的「頁面大小調整和處理」:

- **「縮小超過大小的頁面」/「符合」** → 會縮放(常見 96%),而且**驅動會重新取樣影像**,可能產生顆粒感;券的實際尺寸也會跟著縮(45mm × 96% = 43.2mm,不是你要的 45mm)。
- **「實際大小」** → 100%,不重新取樣。

**注意頁面尺寸不同的兩份檔案會被套用不同的縮放比例**——例如 Canva 匯出的頁面含出血是 222.1×309.1mm(比 A4 大),會被縮到約 90%;自製的 210×297mm 只縮到 96%。**這兩份的比較從一開始就不公平。**

## 什麼時候才真的需要 CMYK

- **家用/辦公室印表機** → 送 **RGB**。若驅動是 PCL,CMYK 只會更糟。
- **印刷廠** → 送 **CMYK + 對方指定的 ICC + OutputIntent**。印刷廠的 RIP 是 PostScript,那時候這些才會生效,而且他們會要求。

## 這次的完整教訓

我當時依序嘗試並「驗證通過」的東西:

| 做了什麼 | 驗證結果 | 對實際列印的效果 |
|---|---|---|
| 換成參考檔同款 ICC(GRACoL 2013 CRPC6) | CMYK 色值比對到 **0 差** | **無** |
| 補上 OutputIntent | 確認寫入 | **無** |
| PDF 1.3 → 1.7 | 檔頭確認 | **無** |
| 換影像編碼 / 解析度 | 4 種管線都測 | **無** |
| 直接用參考檔的 CMYK 像素當底圖 | 像素 100% 相同 | **無** |

**每一項都「技術上正確」,每一項都毫無作用**——因為那條路根本沒被走到。

**教訓:當你把某個變因調到「完全相同」卻沒有任何效果時,不要繼續往下調——
那代表你調的東西根本沒生效。停下來,去確認那條路徑有沒有被執行。**
