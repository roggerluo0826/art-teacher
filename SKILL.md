# 美術小老師

美術老師已經把券畫好了,你的工作是**把它印很多份、每份編號不同**。原圖的版面、配色、字體、比例一個像素都不能動,只有編號那幾個字換掉。

**你不是設計師,你是印刷廠。**

## 什麼時候不要用這個技能

- **圖還沒定稿** → 先請使用者定稿。編號拼版是最後一步,圖一改就要全部重來。
- **使用者要的是「在 Canva/Illustrator 裡面」把版面排滿** → 這個技能產出的是 PDF,不會回頭去動原始設計檔。若使用者堅持要留在 Canva,那是 Canva 的「大量建立(Bulk Create)」功能,不是這裡(注意 Bulk Create 是「一列資料 = 一頁」,要每頁 12 張就得把 CSV 做成 12 欄 × N 列)。
- **編號底下不是單一純色** → 見 `references/pitfalls.md`,先跟使用者討論,不要硬塗。

## 動手前必問

1. **編號規則**:前綴、位數、起始號、總數(例:`HMS` / 3 位 / 從 1 / 共 150)
2. **每頁幾張、幾欄幾列**(例:2 欄 × 6 列 = 12 張)
3. **紙張**(預設 A4)

然後**主動把頁數算給使用者看**,不要等他發現:

> 150 ÷ 12 = 12.5 → 共 **13 頁**,前 12 頁滿版(HMS001–HMS144),第 13 頁只有 **6 張**(HMS145–HMS150),空 6 格。

「總數不是每頁張數的倍數」是最常見的誤會,一定要先講清楚再動手。

## 環境(先讀,否則你會浪費很多時間)

- **用 `py`,不要用 `python`。** PATH 上的 `python` 通常是 Microsoft Store 的 App Execution Alias stub,執行後**沒有任何輸出、回 exit 9009 或 49**,會讓你以為套件壞了、程式炸了,實際上根本沒跑到真的 Python。`py --version` 有正常輸出才是對的。
- 繁中 Windows 先 `$env:PYTHONUTF8=1`,否則 cp950 會炸。
- 需要:`pillow` `numpy` `reportlab` `pymupdf`。
- 路徑含中文時用 PowerShell 帶完整路徑;**檔案「找不到」時先 `Get-ChildItem` 列出資料夾實際內容**,不要假設檔名沒變(使用者可能已經改名或重新匯出)。

## 四個步驟

### 第 1 步:讀圖

```
py scripts/inspect_art.py <圖檔> --text HMS001 --color FF3131 --region top-right --out inspect.json
```

`--region` **一定要限縮**。編號是紅色,圖上別的東西(貨櫃、logo、裝飾)很可能也是紅色,不限縮就會把 bbox 撐成整張圖。腳本若印出「bbox 幾乎等於搜尋範圍寬度」就是抓錯了。

輸出的 `mask_ring_uniform` 是**放行條件**:
- `true` → 編號壓在單一純色上(例:白色雲朵),可以無痕塗掉,繼續。
- `false` → **停下來**,不要硬塗,會露出補丁。見 `references/pitfalls.md`。

### 第 2 步:配字體

```
py scripts/match_font.py inspect.json --out font.json --overlay overlay.png
```

原圖的字體通常不在電腦裡(Canva 用雲端字體)。腳本用**像素疊合 IoU** 客觀挑最接近的系統字型,並算出字級/字距/基線。

- **不要用肉眼猜字體。** IoU 會抓到肉眼會忽略的差異——例如某些字體的「1」有底座橫畫、有些沒有,這一眼很難看出但印出來就是不對。
- **一定要用 Read 看 `overlay.png`**(紅=原圖獨有、綠=候選獨有、黃=重合)。黃色越滿越好。
- IoU < 0.6 → 字形差異明顯,告訴使用者,並問要不要裝原字體(Canva 常用 Poppins / Montserrat,Google Fonts 免費)後用 `--fonts` 指定。
- 腳本會檢查**數字是否等寬**。等寬(如 Arial)→ 所有編號寬度相同,可安心左對齊;不等寬 → 不同編號會長短不一,要回報使用者。

### 第 3 步:拼版

```
py scripts/impose.py inspect.json font.json --prefix HMS --digits 3 --start 1 --count 150 \
   --cols 2 --rows 6 --out out.pdf
```

- 券的長寬比**一律沿用原圖**,絕不拉伸變形。所以拼版後上下通常會多出留白,這是對的。
- 底圖只嵌入 PDF **一次**(同一個 `ImageReader` 重複引用),編號用**向量文字**畫。150 張券因此只有 0.2MB 而非數百 MB,且編號放到多大都銳利。
- 腳本會回報**有效 DPI**,低於 300 要告訴使用者原圖解析度不足。

### 第 4 步:驗證(不可略過)

```
py scripts/verify_pdf.py out.pdf --prefix HMS --digits 3 --start 1 --count 150 \
   --cols 2 --rows 6 --proof proof
```

從產出的 PDF **把編號抽回來**比對:無重複、無缺漏、順序正確、欄列等距、樣式一致、底圖只嵌一次。必須 `exit 0`。

**然後一定要用 Read 打開 `proof_first.png` 和 `proof_last.png` 親眼看過。**
腳本說通過 ≠ 印出來是對的。數字對、位置錯,驗證腳本抓不到。

## 回報給使用者時要講的

- 頁數 / 總張數 / **末頁幾張**
- 單張實際尺寸(mm)與**有效 DPI**
- **選了哪個字體、IoU 多少**——要誠實說「這不是原字體,是最接近的替代」
- 附上 proof 預覽圖

## 鐵則

1. **絕不變形原圖。** 長寬比一律沿用原圖,寧可留白。
2. **遮罩外框不是單一色就停。** 不要賭。
3. **沒親眼看過 proof 圖,不准說「好了」。**
4. **字體是「最接近」不是「原本的」**,講明白,不要讓使用者以為是原字體。
5. **不要試圖在 Canva UI 上自動化編輯 150 次。** Canva 是 canvas 應用,版面常是「表格 + 儲存格內的圖」,逐格改編號又慢又容易錯位。要嘛用它內建的大量建立,要嘛就用這個技能產 PDF。
